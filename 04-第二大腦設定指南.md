# Claude Code 懶人包 #04：第二大腦設定指南

> 版本：v1.0
> 更新日期：2026-04-06
> 對應影片：Claude基本功 EP08
> 前置條件：已完成懶人包 #03（Obsidian 已連接 Claude Code）

> 📌 **這份說明書是給 Claude Code 讀的**。你只需要把這份 MD 檔丟給 Claude Code 桌面版，它會自動幫你建好完整的第二大腦結構。

---

## 這個懶人包會幫你做什麼？

把你剛裝好的空 Obsidian 筆記庫，變成一個完整的 AI 知識管理系統：

- 建好三層資料夾結構（輸入 → 消化 → 輸出）
- 設定好 Claude 的工作規則（CLAUDE.md）
- 建好常用的筆記模板
- 設定每週自動知識重整排程
- 教你怎麼用 Web Clipper 收集資料

完成後，你只要做兩件事：
1. **平時**：看到好文章 → Web Clipper 存到 Clippings
2. **每週日**：Claude 自動幫你整理成知識庫

---

## 請 Claude Code 幫我執行以下步驟

> ⚠️ 以下內容是給 Claude Code 讀的操作指令。
> 你只需要把這整份 MD 檔丟給 Claude Code 桌面版，它會自動開始執行。
> 遇到需要你手動操作的地方，它會暫停並告訴你該怎麼做。

---

## 階段一：認識你

> 在開始之前，Claude 需要先了解你，才能幫你客製化第二大腦。

### 步驟一：詢問使用者基本資訊

> 🖐️ **需要使用者回答**：請依序詢問以下問題，等使用者回答後再繼續。

1. 你教什麼**科目**？（例如：國中數學、高中英文、國小自然）
2. 你教什麼**年級**？（例如：七年級、高一）
3. 你希望 Claude 用什麼**語言**回答？（預設：繁體中文）
4. 你有沒有其他偏好？（例如：回答要簡潔、要附教學建議、要用白話文）

> 記下使用者的回答，後續步驟會用到。

---

## 階段二：建立三層資料夾結構

### 步驟二：建立資料夾

這個結構的靈感來自 AI 大神 Karpathy 的 AI 筆記流：

```
vault/
├── Clippings/           ← 輸入：你在網路上看到的好文章、影片筆記
├── 知識庫/              ← 消化：AI 幫你整理後的結構化知識
├── 創作庫/              ← 輸出：你自己的原創內容（教案、教材、報告）
├── 每日筆記/            ← 時間管理：每天和每週的紀錄
├── Templates/           ← 模板：各種筆記的固定格式
└── CLAUDE.md            ← Claude 的班規：告訴 AI 你是誰、怎麼工作
```

**三層的邏輯：**

| 層 | 資料夾 | 白話說法 | 誰寫的 |
|---|---|---|---|
| 輸入 | `Clippings/` | 你在網路上「剪」下來的東西 | 別人的文章，你只是收集 |
| 消化 | `知識庫/` | AI 幫你整理成「百科全書」 | AI 寫，你審核 |
| 輸出 | `創作庫/` | 你自己的作品和教材 | 你寫的，你的智慧結晶 |

請在使用者的 Obsidian vault 中建立以上所有資料夾。
如果某些資料夾已存在，跳過不覆蓋。

---

## 階段三：建立 CLAUDE.md

### 步驟三：在 vault 根目錄建立 CLAUDE.md

這是 Claude 每次開對話時都會自動讀取的「班規」。
請根據步驟一使用者的回答，建立以下內容：

```markdown
# 我的第二大腦 — CLAUDE.md

## 關於我
- 我是[使用者回答的科目][使用者回答的年級]老師
- 這個 vault 是我的教學第二大腦

## 語言偏好
- 所有回應請使用[使用者回答的語言]
- 筆記內容也用[使用者回答的語言]

## 筆記庫結構（三層）

| 資料夾 | 用途 | 存什麼 |
|---|---|---|
| `Clippings/` | 輸入 | 網路上剪藏的文章、影片筆記、別人的資料 |
| `知識庫/` | 消化 | AI 整理後的結構化知識（有 index.md 索引） |
| `創作庫/` | 輸出 | 我自己的教案、教材、報告、原創內容 |
| `每日筆記/` | 時間管理 | 每日紀錄、週計畫、知識重整週報 |
| `Templates/` | 模板 | 各種筆記的固定格式 |

## 工作規則

### 新增筆記時
- 一律加上 frontmatter（title、date、tags）
- 根據內容判斷存到哪個資料夾：
  - 別人的文章 → `Clippings/`
  - AI 整理的知識 → `知識庫/`
  - 我自己的作品 → `創作庫/`
  - 每日/每週紀錄 → `每日筆記/`

### Clippings 處理規則
- 使用者透過 Web Clipper 剪藏的文章會自動存到 `Clippings/`
- 這些是原始資料，**不要修改**
- 每週知識重整時，AI 會把 Clippings 消化成知識庫頁面

### 知識庫維護規則
- 知識庫有 `index.md`（目錄）和 `log.md`（操作紀錄）
- 每次新增知識庫頁面，都要更新 index.md
- 每次做知識重整，都要在 log.md 新增紀錄

### 使用者說「幫我新增到筆記」時
- 代表存到這個 Obsidian vault
- 根據內容判斷存到正確的資料夾
```

> 把 `[使用者回答的科目]`、`[使用者回答的年級]`、`[使用者回答的語言]` 替換為步驟一的實際回答。

## 階段四：建立 Templates

### 步驟四：建立筆記模板

```markdown
type: 每日筆記
tags:
  - 每日筆記
---

# <% tp.file.title %>

---

## 🏫 教學

-

**備課進度**：

---

## 💡 今日反思

>

---

1.
2.
3.
```

```markdown

week: <% tp.date.now("YYYY-[W]WW") %>
```

## 階段六：設定 Web Clipper

### 步驟六：引導使用者設定 Web Clipper 預設資料夾

> 🖐️ **需要手動操作**：請告知使用者以下步驟。

你已經安裝了 Obsidian Web Clipper，現在要把它的預設存放位置改成 `Clippings/`：

1. 打開瀏覽器，點選 Web Clipper 的圖示（通常在網址列右邊）
2. 點選齒輪 ⚙ 進入設定
3. 找到「Vault」，確認選的是你的 vault
4. 找到「Folder」或「Location」（存放位置）
5. 改成 `Clippings`
6. 儲存設定

設定完成後，以後你在網頁上按 Web Clipper，文章就會自動存到 `Clippings/` 資料夾。

> **為什麼要存到 Clippings？**
> 因為這些是「別人的文章」，屬於原始輸入。每週知識重整時，AI 會自動把這些文章消化成你的知識庫。你不需要自己整理，先存再說。

## 階段七：設定每週知識重整排程

### 步驟七：建立自動排程

使用 Claude Code 的排程功能，建立一個每週日自動執行的知識重整任務。

請建立以下 scheduled task：

- **任務 ID**：`weekly-knowledge-review`
- **排程**：每週日早上 9:17（cron：`17 9 * * 0`）
- **說明**：每週日自動知識重整（Karpathy 式七步驟）

任務內容（prompt）如下：

```
請執行 Obsidian 筆記庫的每週知識重整，完成以下所有步驟：

第一步：盤點本週變動
- 搜尋 vault 中過去 7 天內新增或修改的筆記
- 分三類列出：Clippings/（外部輸入）、創作庫/（原創內容）、知識庫/（已有的知識頁面）

第二步：消化 Clippings → 知識庫
- 讀取 Clippings/ 中本週新增的所有筆記完整內容
- 分析主題、找出跨篇關聯
- 為每個主題群產出一篇結構化的知識庫頁面（含 frontmatter、核心概念、連結、缺口清單）
- 存到知識庫/
- 如果本週沒有新的 Clippings，跳過此步驟

第三步：創作庫回流
- 讀取創作庫/ 中本週新增或修改的筆記
- 檢查是否有新的概念值得提煉到知識庫
- 如果有，更新對應的知識庫頁面

第四步：知識庫健康檢查（lint）
- 連結檢查：找出可以互相連結但尚未連結的筆記
- 知識缺口：根據主題分佈，建議該去找什麼資料
- 孤兒頁面：找出沒有任何連結的筆記

第五步：更新 index.md 和 log.md
- index.md：加入本週新增的知識庫頁面
- log.md：新增本週重整紀錄

第六步：產出週報筆記
- 存到每日筆記/ 資料夾
- 檔名：YYYY-WXX 知識重整.md
- 內容：本週變動總覽、消化了什麼、健康檢查結果、下週建議

完成後通知使用者：「本週知識重整已完成，請查看週報筆記。」
```

> 建立完成後，告知使用者：
> 「✅ 排程已設定！每週日早上，Claude 會自動幫你做知識重整。」
> 「你也可以隨時說『跑一次知識重整』來手動觸發。」

## 階段八：建立歡迎筆記 + 驗證

### 步驟八：建立第一篇筆記

在 vault 根目錄建立一篇歡迎筆記，內容如下：

```markdown
---
title: 歡迎來到你的第二大腦
date: [今天日期]
type: 指南
tags:
  - 指南
---

# 🧠 歡迎來到你的第二大腦！

恭喜！你的 AI 知識管理系統已經設定完成。

---

## 你的筆記庫結構

| 資料夾 | 用途 | 你要做的事 |
|---|---|---|
| `Clippings/` | 輸入 | 看到好文章 → 按 Web Clipper → 自動存進來 |
| `知識庫/` | 消化 | 不用管，每週日 AI 會自動整理 |
| `創作庫/` | 輸出 | 你自己的教案、教材、報告放這裡 |
| `每日筆記/` | 時間管理 | 每天的紀錄、每週的計畫 |

---

## 你每天只要做兩件事

### 1. 收集（10 秒）
看到好文章 → 按 Web Clipper → 完成。不用整理，先存再說。

### 2. 收割（需要的時候）
跟 Claude 說「幫我從筆記庫整理出 XXX」，AI 會幫你從所有筆記中串接出你需要的文件。

---

## 每週自動發生的事

每週日早上，Claude 會自動執行知識重整：
1. 把本週的 Clippings 消化成知識庫
2. 檢查知識庫的健康狀態
3. 產出一份週報放在每日筆記/

你不用做任何事，打開 Obsidian 就能看到結果。

---

## 常用指令

| 你說的話 | Claude 會做的事 |
|---|---|
| 「幫我新增到筆記」 | 把內容存到 Obsidian（自動判斷資料夾） |
| 「搜尋筆記有沒有 XXX」 | 搜尋整個 vault |
| 「跑一次知識重整」 | 手動觸發完整七步驟 |
| 「消化 Clippings」 | 只消化本週新增的剪藏文章 |
| 「知識庫 lint」 | 只做健康檢查 |
| 「幫我寫今天的教學反思」 | 引導你口述，整理成筆記 |

---

## 開始你的第一步

現在就試試看：
1. 打開瀏覽器，找一篇你覺得不錯的文章
2. 按 Web Clipper 存到 Clippings
3. 回到 Claude Code，說「消化 Clippings」
4. 看看知識庫裡多了什麼！
```

全部通過後，告知使用者：
